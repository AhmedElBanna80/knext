## Live findings from today's OKE test (fold these into Tier A)
- **Control-plane consolidation is the #1 gap (ADR-0001).** The app runs as a RAW `ksvc`; the
  deploy, ingress class, `min-scale`, image, and bytecode-cache volume were ALL hand-patched on a
  live cluster. The operator/`NextApp` CR must own: ksvc reconciliation, correct ingress class at
  install, `min-scale:0` default, **digest-pinned** image (no mutable tags), and a **persistent PVC
  mount for `NODE_COMPILE_CACHE`** (without persistence, bytecode caching only helps intra-pod, not
  across cold starts — the actual goal).
- **Test-coverage gap:** the Dockerfile `CMD` runtime path had ZERO tests; a bug (un-exported
  `NODE_COMPILE_CACHE`, fixed in `f100deb`) shipped through CI + unit tests + 2 reviews and was only
  caught by live test. Maturity needs container/runtime-level assertions, not just unit tests.
- **Ingress correctness:** the `kourier.ingress.networking.knative.dev` class must be set by the
  install/operator, never hand-fixed (today's IngressNotConfigured root cause).
- Scale-to-zero + bytecode caching are the **two non-negotiable core features** — every item must
  preserve them; add regression coverage for both.
