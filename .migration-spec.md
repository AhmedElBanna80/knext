# Ticket: Complete the vinext → official Next.js Adapter migration (code + docs)

## Goal (locked)
Migrate the **framework** to **Node.js with bytecode caching**, on the **official Next.js
Deployment Adapter API + `output:'standalone'`**. Remove all Vinext/Nitro. Update stale docs to
match. The app (`apps/file-manager/next-adapter.ts`) is already migrated — verify, don't regress it.

**Target = Node, not Bun.** Bytecode = **`NODE_COMPILE_CACHE`** (V8 compile cache persisted across
cold starts), set on the Node runtime that runs `server.js` — already wired in the operator
(`packages/kn-next-operator/.../nextapp_controller.go:201`); ensure the adapter/Dockerfile/runtime
set it to a persistent dir and that it actually takes effect. Do NOT use `bun --compile`/`--bytecode`
(rejected — see `docs/research/adapter-bun-learnings.md`: even the official reference Bun adapter
ships no bytecode; Node uses `NODE_COMPILE_CACHE`).

## Stop condition (definition of done — STOP here)
1. All **code** migrated (no active vinext/nitro path) AND **docs** migrated.
2. Node runtime demonstrably uses `NODE_COMPILE_CACHE` (test/asserted).
3. Clean-checkout `pnpm -r build` + `biome check .` (0 errors) + `vitest run` all green.
4. **spec-review AND code-review both APPROVED.**
5. Branch `feat/ADAPTER-MIGRATE` **pushed** and a **PR opened** against `main`.
Push authorization: feature-branch push is now ALLOWED by policy (`git push -u origin
feat/ADAPTER-MIGRATE`, `gh pr create --base main`). NEVER push to main, force-push, or rewrite
history (the hook blocks these).

## Hard rules (.claude/rules/architecture.md + CLAUDE.md — read them)

  `experimental.adapterPath`). Do NOT reverse-engineer or reintroduce Nitro/Vinext.
- **Don't rewrite the runtime twice** — one standalone Node server, runs on Node and Bun.
- **Operator = single source of truth** for cluster state (ADR-0001). Do not add new
  direct-cluster-mutation paths.
- **TDD: RED before GREEN** (enforced by the superteam commit hook). Tests first.
- No secrets in files; no unauthenticated mutating endpoints. Commit prefixes that pass the hook:
  plain `test:`/`feat:`/`chore:`/`docs:`/`ci:` (NO scope parens, e.g. `chore(x):` is REJECTED).
